**Flow** is a multi-tenant platform — each client (Suja's Kitchen, TFLC,
Carmate, Picotin, …) runs its own deployment of the same codebase, set apart by
a single `tenant` tag. These are the cross-cutting conventions every Flow tenant
deployment follows. Two matter day-to-day:

<CardGroup cols={2}>
  <Card title="PostHog usage" icon="chart-simple" href="/conventions/posthog">
    One shared PostHog project, filtered per client by a `tenant` tag.
  </Card>
  <Card title="Workflow codes (W01…)" icon="hashtag" href="/conventions/workflow-codes">
    Billable sidebar sections carry a stable `W0x` code sent on every nav click.
  </Card>
</CardGroup>

<Note>
  These two conventions are linked: the **workflow code** rides on the
  **PostHog** `nav_clicked` event. Read PostHog first.
</Note>
