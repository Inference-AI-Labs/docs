These docs are maintained by a pair of Claude Code skills. One **writes** to the
knowledge base, the other **reads** from it. Together they're how Flow's
engineering knowledge gets in — and back out.

## /write-mint — author a page

`/write-mint <topic>` (or bare, then it asks). It does the whole job, not just
the prose:

<Steps>
  <Step title="Interview">
    Asks only what it can't infer: what the thing is and who it's for, why it
    exists, the actual steps or shape, and the gotchas. When documenting code it
    **reads the source** rather than asking you to recite it — names, flags and
    paths are verified, never invented.
  </Step>
  <Step title="Place it">
    Picks the smallest correct home — extend an existing page, add a sibling page
    to an existing group, or (only for a genuinely new area, with your sign-off)
    a new nav group. Confirms the group + slug.
  </Step>
  <Step title="Write it">
    House-style MDX: `title` + `sidebarTitle` + `description` frontmatter,
    Mintlify components (`Card`, `Note`, `Warning`, `Steps`), filename-labelled
    code blocks, the multi-tenant Flow voice, cross-links. Secrets are always
    redacted to their names.
  </Step>
  <Step title="Validate">
    Parses `docs.json` and runs `mint broken-links`; both must pass before it
    ships.
  </Step>
  <Step title="Ship">
    Shows a one-line summary, then — after a single confirmation, because the
    site auto-deploys from `main` — commits and pushes to `main`.
  </Step>
</Steps>

## /read-mint — answer from the docs

`/read-mint <question>` (e.g. *how does lead-listener auth work?*). It searches
the pages, reads the relevant ones, and answers **only from what's written
here** — quoting exact values and citing the page + live URL. If the docs don't
cover it, it says so plainly and offers to capture it with `/write-mint`. It
won't fill the gap from memory or the codebase.

## House rules they follow

| Rule | Why |
| --- | --- |
| `sidebarTitle` on every long title | Keeps the sidebar single-line |
| Secret **names** only, values redacted | Never leak keys into the docs |
| Match an existing page's structure | Consistent voice across the site |
| One page = one thing | Two topics → two pages |
| `/write-mint` confirms placement + the push | Placement is a nav change; the push deploys |
| `/read-mint` cites every claim, never invents | The docs are the authority, not the model |

<Note>
  `/write-mint` is the canonical way to add to these docs. If you're writing a
  page by hand instead, mirror the [Workers](/workers/overview) pages and the
  rules above — especially `sidebarTitle` and secret redaction.
</Note>
